## Phase 1: MVP — 팀 워크스페이스 + 커스텀 템플릿

### 목표
현재 개인 도구를 팀이 함께 쓸 수 있는 B2B 제품으로 전환. 워크스페이스를 만들고, 분석 결과를 저장하고, 팀원과 공유하며, 커스텀 템플릿으로 변환.

### 기능
- [ ] Supabase 연동 (DB + Storage)
- [ ] 워크스페이스 생성/참여 (접근 코드 방식)
- [ ] 멤버 관리 (닉네임 + 역할)
- [ ] 프로젝트(폴더) CRUD
- [ ] 분석 결과 DB 저장 + 목록/상세 조회
- [ ] 댓글/피드백 기능
- [ ] 커스텀 템플릿 생성/저장/적용
- [ ] 워크스페이스 대시보드 (최근 분석, 프로젝트 목록)

### 데이터
- Workspace, Member, Project, Analysis, Comment, Template

### 인증
- 로그인 없음. 접근 코드로 워크스페이스 입장, 닉네임으로 식별

### "진짜 제품" 체크리스트
- [ ] Supabase 실제 DB 연결 (목업 데이터 X)
- [ ] 접근 코드 기반 실제 워크스페이스 (하드코딩 X)
- [ ] Vercel에 배포 (localhost X)
- [ ] 다른 사람이 URL + 접근 코드로 접속해서 써볼 수 있음

### Phase 1 시작 프롬프트

- 프로젝트 CRUD
- 분석 결과 저장 + 조회
- 댓글 기능
- 커스텀 템플릿 생성/저장/적용
- 대시보드

반드시 지켜야 할 것:
- 04_PROJECT_SPEC.md의 "절대 하지 마" 목록 준수
- Supabase 실제 DB 연결 (목업 X)

## Phase 2: 확장 — 다중 소스 + 배치 처리

### 전제 조건
- Phase 1이 안정적으로 배포된 상태

### 목표
YouTube 외 소스(파일 업로드, 웹페이지)도 분석 가능하게 확장. 여러 URL을 한번에 분석하는 배치 기능 추가.

### 기능
- [ ] 파일 업로드 분석 (MP4/MP3/WAV → Whisper API로 자막 추출)
- [ ] 웹페이지 URL 분석 (텍스트 추출 → AI 요약)
- [ ] 배치 분석 (여러 URL 큐에 넣고 순차 처리)
- [ ] 분석 히스토리 검색 + 필터
- [ ] 태그 기능 (분석 결과 분류용)

### 추가 데이터
- Analysis에 `source_type` 필드 추가 ("youtube" / "file" / "webpage")
- Tag 엔티티 추가

### 통합 테스트
- Phase 1 기능(워크스페이스, 프로젝트, 댓글, 템플릿)이 여전히 정상 동작하는지 확인

## Phase 3: 고도화 — 분석 + 대시보드

### 전제 조건
- Phase 1 + 2가 안정적으로 운영 중

### 목표
경쟁 채널 자동 모니터링, 트렌드 키워드 대시보드, API 제공으로 외부 툴과 연동.

### 기능
- [ ] 채널 등록 → 신규 영상 자동 분석
- [ ] 키워드/트렌드 대시보드 (분석 결과 종합)
- [ ] REST API 제공 (외부 연동용)
- [ ] 관리자 패널 (사용량 통계, 워크스페이스 관리)
- [ ] 내보내기 확장 (Notion, Google Docs 연동)

### 주의사항
- 자동 수집은 YouTube API 할당량과 비용 고려 필요
- Whisper API, Claude API 비용이 사용량에 비례하여 증가
- API 제공 시 rate limiting 필수
